# Growth Content Execution Checklist

Version: 1.0  
Owner: Growth + Product + Content  
Last updated: 2026-02-20  
Review cadence: Monthly

Use this checklist for every content sprint and page shipment.

## A) Pre-Brief

- [ ] Primary persona selected
- [ ] Funnel stage selected
- [ ] Primary search intent selected
- [ ] Main KPI and guardrail KPI selected
- [ ] Primary CTA selected

## B) Draft Requirements

- [ ] Opening gives direct answer in 40-60 words
- [ ] Query-style H2/H3 headings used
- [ ] Claim -> mechanism -> evidence used where persuasive
- [ ] Clear next-step action included
- [ ] Canonical terms used (`AISkillScore`, `tokens`)

## C) Brand + Trust QA

- [ ] No overclaims or guarantees
- [ ] No banned pricing language (`cheap`, `budget hack`)
- [ ] No hallucinated facts about user background
- [ ] Voice is practical, grounded, non-hype

## D) SEO/GEO QA

- [ ] Primary keyword appears naturally in title/H1/intro
- [ ] Supporting terms used without stuffing
- [ ] FAQ section included for high-intent pages
- [ ] Internal links to tool/pricing/related page included
- [ ] Metadata and schema requirements tracked for publish

## E) Monetization QA (When Applicable)

- [ ] Outcome-first value framing precedes price mention
- [ ] Top-up path included for shortfall scenarios
- [ ] Larger-pack value anchor included
- [ ] Token math is explicit and transparent
- [ ] Trust reassurance included (token safety)

## F) Publish + Measurement

- [ ] Publish date and owner logged
- [ ] Hypothesis and expected KPI impact logged
- [ ] 7-day review task created
- [ ] 30-day optimization task created
- [ ] Learnings written back into playbook/templates
